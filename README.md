## Быстрое описание

**ArmaLauncher** — это лаунчер, который помогает игроку без лишней возни:
- установить игру или нужную сборку;
- скачать и обновить моды;
- проверить целостность файлов;
- быстро подключиться к нужному серверу;
- видеть новости, состояние серверов и прогресс загрузки в одном окне.

Если говорить совсем просто: это программа, которая делает вход в проект **понятным, быстрым и красивым**, даже если человек не хочет разбираться в технических деталях.

## Что умеет лаунчер

- Установка и обновление игры
- Установка и обновление модов
- Проверка файлов на повреждение
- Поддержка нескольких серверов
- Мониторинг серверов и отображение статуса
- Новости внутри лаунчера

- Диагностика и проверка скорости соединения
- Красивый интерфейс с фоновыми изображениями
- Поддержка локализации
- Быстрая загрузка больших файлов с оптимизацией скачивания

## Для обычного пользователя

### Что это вообще такое?
Это лаунчер для игрового проекта на базе **Arma Reforger**. Он нужен, чтобы игроку не приходилось вручную искать моды, перекачивать файлы, разбираться где что лежит и почему что-то не запускается.

### Что делает за тебя?
1. Подключается к серверу проекта
2. Проверяет, что установлено
3. Докачивает недостающее
4. Обновляет устаревшие файлы
5. Даёт запустить игру с нужной конфигурацией

### Почему это удобно?
- меньше ошибок при установке;
- не нужно вручную возиться с кучей файлов;
- проще поддерживать актуальную сборку;
- удобно для игроков, которые вообще не шарят в техничке.

## Быстрый старт

### Клиент
1. Открой `ArmaLauncherClient`
2. Запусти приложение
3. Укажи путь к игре и модам при необходимости
4. Выбери сервер
5. Нажми установку / обновление
6. После завершения запускай игру

### Сервер
1. Открой `LauncherServer`
2. Заполни конфиг `appsettings.json`
3. Укажи пути к игре, моделям, аддонам и чанкам
4. Запусти сервер
5. Подключи клиент к `BaseUrl`

## Архитектура проекта

Проект разделён на две основные части:

### `ArmaLauncherClient`
Клиентская часть на **WPF**.

Отвечает за:
- интерфейс;
- отображение новостей;
- управление загрузками;
- выбор сервера;
- проверку файлов;
- запуск игры;
- отображение прогресса, скорости и статусов.

### `LauncherServer`
Серверная часть на **ASP.NET Core**.

Отвечает за:
- выдачу информации об игре и модах;
- список моделей и аддонов;
- раздачу файлов;
- раздачу чанков;
- подготовку контента;
- кэширование и ускорение скачивания;
- API для клиента.

## Расширенное техническое описание

### Как работает клиент
Клиент использует `UpdateManager`, который управляет:
- путями установки игры и модов;
- загрузкой файлов;
- проверкой уже существующих данных;
- прогрессом скачивания;
- режимами загрузки;
- отменой операций;
- сохранением локальных настроек.

В проекте есть как минимум два режима загрузки:
- **Phased** — сначала скачивание, потом сборка; максимальная скорость, но выше расход памяти;
- **Streaming** — скачивание и запись параллельно; ниже расход памяти.

Дополнительно клиент поддерживает:
- дедупликацию чанков;
- верификацию файлов;
- отображение ETA, скорости и текущего файла;
- выбор языка;
- смену разделов интерфейса;
- слайдшоу фоновых изображений.

### Как работает сервер
Сервер предоставляет HTTP API для лаунчера.

Основные endpoints:
- `/health` — проверка состояния сервера;
- `/info` — общая информация;
- `/game` — информация по игре;
- `/models` — список моделей;
- `/models/{modelId}` — детали модели;
- `/addons` — список аддонов;
- `/addons/{folderId}` — детали аддона;
- `/addons/{folderId}/files/{**filePath}` — выдача файла;

Сервер также:
- поддерживает компрессию ответов;
- умеет работать с range-запросами;
- ограничивает запросы по rate limit;
- логирует запуск и действия;
- работает с локальным хранилищем данных.

## Система чанков и быстрой раздачи

Одна из ключевых фишек проекта — работа с большими файлами через чанки.

### Что это даёт?
- не нужно перекачивать всё целиком;
- можно переиспользовать уже известные куски данных;
- ускоряется доставка обновлений;
- уменьшается общий объём скачивания.

### Подготовка контента
Серверная CLI-утилита умеет:
- разбивать файлы или папки на чанки;
- сжимать чанки;
- создавать manifest;
- подписывать manifest;
- экспортировать публичный ключ.

Это позволяет строить безопасную и быструю систему обновлений.

---

## Edge / CDN / Cloudflare

В проекте предусмотрен worker для edge-кэширования.

Он используется для:
- кэширования manifest-файлов;
- выдачи чанков через edge;
- выдачи patch-файлов;
- работы с публичным ключом;
- ускорения доступа к данным через Cloudflare.

Если нужен масштабируемый деплой — это сильная часть архитектуры.
